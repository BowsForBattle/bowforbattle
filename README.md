# Bows for Battle Website

This guide explains how to download and open the website if you are not a programmer.

## 1-Minute Quick Start

1. Open the GitHub project page.
2. Click **Code** -> **Download ZIP**.
3. Extract the ZIP file.
4. Open the extracted folder.
5. Double-click `index.html`.

If images are missing, make sure the `Pictures` folder is still inside the same folder as `index.html`.

## What You Need

- A computer with internet access
- A web browser (Chrome, Edge, Firefox, or Safari)

## Option 1: Download as a ZIP (Easiest)

1. Go to the project page on GitHub.
2. Click the green **Code** button.
3. Click **Download ZIP**.
4. Find the downloaded ZIP file on your computer (usually in Downloads).
5. Right-click the ZIP file and choose **Extract All**.
6. Open the extracted folder.
7. Double-click `index.html`.

The site should now open in your browser.

## Option 2: Download with GitHub Desktop

Use this option if someone helping you already installed GitHub Desktop.

1. Open GitHub Desktop.
2. Click **File > Clone Repository**.
3. Choose the **URL** tab.
4. Paste the repository URL.
5. Choose where to save it on your computer.
6. Click **Clone**.
7. Open the downloaded folder.
8. Double-click `index.html`.

## How to Open Other Pages

After opening the home page (`index.html`), use the top navigation menu to visit:

- Mission
- Programs
- Support
- Contact

You can also open these files directly from the folder:

- `mission.html`
- `programs.html`
- `support.html`
- `contact.html`

## Important Folder Note

Keep the `Pictures` folder in the same location as the HTML files.

If the `Pictures` folder is missing or moved, images may not load.

## If Something Looks Broken

Try these quick fixes:

1. Make sure all files were extracted from the ZIP.
2. Confirm `index.html` and the `Pictures` folder are in the same main folder.
3. Refresh the page in your browser.
4. Try a different browser.

## Sharing the Site

To share this website with someone else:

1. Share the full extracted folder (not only one HTML file).
2. Include the `Pictures` folder.

That ensures the site keeps its styling and images.
